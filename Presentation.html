This project is a classic Snake game built using Python's Turtle graphics library. The game features real-time gameplay, score tracking, high score persistence, and complete collision detection. It's an excellent example of game development using simple graphics and event-driven programming.

Game Features:

  • Collision detection (borders and self)
  • Smooth animation with configurable game speed

üéÆ Snake Game (snake.py)

1. Setup & Configuration

The game begins by defining game constants and initializing the turtle graphics environment.

WIDTH, HEIGHT = 600, 600 DELAY = 0.12 MOVE_STEP = 20 BOUNDARY = (WIDTH // 2) - 10 delay = DELAY score = 0 high_score = 0 segments = []
What it does: Sets up the game window dimensions (600x600 pixels), controls game speed with DELAY, and defines how far the snake moves per step. The segments list stores the snake's body parts.

2. Game Window & Turtle Objects

Creates the visual game environment and game objects (snake head, food, and score display).

wn = turtle.Screen() wn.title("April's Favorite Snake") wn.bgcolor("black") wn.setup(width=WIDTH, height=HEIGHT) wn.tracer(0) head = turtle.Turtle() head.shape("circle") head.color("white") head.penup() head.goto(0, 0) head.direction = "stop"
What it does: Creates a 600x600 black game window with title "April's Favorite Snake". The wn.tracer(0) disables automatic screen updates for better control. The head object represents the snake's head - a white circle that starts at position (0, 0) with direction set to "stop".

3. Food & Score Display

Initializes the food object and score visualization.

food = turtle.Turtle() food.shape("circle") food.color("red") food.penup() food.goto(0, 100) pen = turtle.Turtle() pen.hideturtle() pen.color("white") pen.penup() pen.goto(0, (HEIGHT // 2) - 40) pen.write(f"Score: {score}", align="left", font=("Courier", 16, "normal"))
What it does: Creates a red circle food object at position (0, 100). The pen object displays the score at the top of the screen. The hideturtle() method hides the pen's cursor while keeping its text visible.

4. Movement Functions

Defines functions that control the snake's direction and movement.

def go_up(): if head.direction != "down": head.direction = "up" def go_down(): if head.direction != "up": head.direction = "down" def go_left(): if head.direction != "right": head.direction = "left" def go_right(): if head.direction != "left": head.direction = "right" def move(): x, y = head.xcor(), head.ycor() if head.direction == "up": head.sety(y + MOVE_STEP) if head.direction == "down": head.sety(y - MOVE_STEP) if head.direction == "left": head.setx(x - MOVE_STEP) if head.direction == "right": head.setx(x + MOVE_STEP)
What it does: The go_* functions change the snake's direction, but with validation - you cannot move directly backward (e.g., cannot go down if currently going up). The move() function updates the head's position based on its current direction by MOVE_STEP pixels.

5. Keyboard Controls

Maps keyboard inputs to the movement functions.

wn.listen() wn.onkey(go_up, "w") wn.onkey(go_down, "s") wn.onkey(go_left, "a") wn.onkey(go_right, "d") wn.onkey(go_up, "Up") wn.onkey(go_down, "Down") wn.onkey(go_left, "Left") wn.onkey(go_right, "Right")
What it does: Activates keyboard listening and binds two sets of controls - WASD keys and Arrow keys. Both control schemes perform the same functions for player convenience.

6. Main Game Loop

The core game logic runs continuously in an infinite loop.

while True: wn.update() # Border collision detection if abs(head.xcor()) > BOUNDARY or abs(head.ycor()) > BOUNDARY: # Reset game head.goto(0, 0) head.direction = "stop" # Hide body segments for seg in segments: seg.goto(1000, 1000) segments.clear() score = 0
What it does: Continuously updates the game window. Detects when the snake hits the border and resets the game - moves the head back to center, hides all body segments by moving them off-screen, clears the segments list, and resets the score.

7. Food Collision & Growth

Handles what happens when the snake eats food.

if head.distance(food) < 20: # Move food to random location max_cell = (BOUNDARY - MOVE_STEP) // MOVE_STEP x = random.randint(-max_cell, max_cell) * MOVE_STEP y = random.randint(-max_cell, max_cell) * MOVE_STEP food.goto(x, y) # Add body segment new_seg = turtle.Turtle() new_seg.speed(0) new_seg.shape("circle") new_seg.color("green") new_seg.penup() segments.append(new_seg) # Increase speed and score delay = max(0.02, delay - 0.005) score += 1
What it does: Checks if the snake's head is within 20 pixels of food. When food is eaten: (1) Food moves to a random grid-aligned position, (2) A green body segment is created and added to the snake, (3) Game speed increases slightly (decreases delay), (4) Score increments. The game gets progressively faster!

8. Body Movement

Makes the body segments follow the head in a "chain" pattern.

for index in range(len(segments) - 1, 0, -1): x = segments[index - 1].xcor() y = segments[index - 1].ycor() segments[index].goto(x, y) if segments: segments[0].goto(head.xcor(), head.ycor())
What it does: Creates the "following" effect - each segment moves to the position of the segment in front of it. Starting from the tail (working backwards), each segment takes the position of the one ahead. The first body segment moves to where the head currently is.

9. Collision with Self

Detects if the snake collides with its own body.

for seg in segments: if seg.distance(head) < 10: time.sleep(0.5) head.goto(0, 0) head.direction = "stop" for s in segments: s.goto(1000, 1000) segments.clear() score = 0 delay = DELAY break
What it does: Checks if the head is within 10 pixels of any body segment. If a collision is detected, the game resets just like hitting a border - pauses briefly, resets position, clears body, and resets score and speed.

üéØ Key Programming Concepts

Object-Oriented Programming

Uses turtle objects (head, food, pen, segments) with properties and methods like .goto(), .color(), and .distance().

Game Loop

The infinite while True loop continuously updates the game state, checks collisions, and re-renders the screen.

Event Handling

Keyboard events are captured using wn.listen() and wn.onkey() to control player input.

Collision Detection

Uses distance calculations and position checks to detect when the snake hits borders, food, or itself.

List Management

The segments list dynamically grows as the snake eats, and iterations update each segment's position.

Randomization

Uses random.randint() to spawn food at random grid-aligned positions on the game board.
